Welcome to our blog! This is the first article demonstrating file-based dynamic routing in Django.

What is file-based routing?

File-based routing is an approach where web page URLs are automatically generated based on the file and folder structure in the project. Instead of manually creating URL patterns for each page, the system determines routes automatically by analyzing the file structure.

Advantages of this approach:

  • Simplicity: No need to write URL patterns for each page
  • Intuitiveness: URL structure reflects file structure
  • Scalability: Easy to add new pages
  • Maintainability: Simple renaming and moving of pages

How it works in our project?

In our Django project we created a system that:

  1. Scans the templates/pages/ folder
  2. Automatically creates URL patterns for all found HTML files
  3. Supports nested folders for content organization
  4. Handles special cases (e.g., index.html as root page)

Structure example:

templates/pages/
├── index.html          → /
├── about.html          → /about/
├── contact.html        → /contact/
└── blog/
    ├── index.html      → /blog/
    ├── first-post.html → /blog/first-post/
    └── article.html    → /blog/article/

Conclusion

File-based routing makes web development more convenient and intuitive. This is especially useful for content websites, blogs, and documentation.
